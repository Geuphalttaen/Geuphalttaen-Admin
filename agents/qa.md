# QA Agent — 급할땐 Admin

## 역할

Playwright E2E 테스트로 급할땐 Admin 웹 앱의 주요 플로우를 검증합니다.

## 담당 범위

- Playwright E2E 테스트 작성 및 실행 (`e2e/` 디렉토리)
- 시나리오별 테스트 케이스 정의
- Critical / High 버그 리포트 작성

## 테스트 환경

- 테스트 대상: `http://localhost:3000`
- 백엔드: 로컬 또는 스테이징 서버 (`http://localhost:8080`)
- 테스트 계정: 환경 변수 `ADMIN_EMAIL`, `ADMIN_PASSWORD` 사용

## E2E 테스트 시나리오

### 시나리오 1: 관리자 로그인

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/login` 접근 | 로그인 폼 표시 |
| 2 | 유효한 이메일 + 비밀번호 입력 후 제출 | `/dashboard` 로 이동 |
| 3 | 잘못된 비밀번호로 로그인 시도 | 에러 메시지 표시, 페이지 이동 없음 |
| 4 | 빈 필드로 제출 | zod 검증 에러 메시지 표시 |
| 5 | 인증 상태로 `/login` 직접 접근 | `/dashboard` 로 리다이렉트 |
| 6 | 미인증 상태로 `/dashboard` 직접 접근 | `/login` 으로 리다이렉트 |

### 시나리오 2: 제보 승인

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/reports` 접근 | 제보 목록 테이블 표시 |
| 2 | PENDING 제보 행의 "승인" 버튼 클릭 | 확인 모달 또는 즉시 처리 |
| 3 | 승인 확인 | 해당 제보의 상태가 ACTIVE로 변경, 목록 갱신 |
| 4 | `/reports/[id]` 상세 접근 | 제보 정보 표시 |
| 5 | 상세 페이지에서 "승인" 버튼 클릭 후 확인 | 상태 ACTIVE 변경, 버튼 비활성화 |

### 시나리오 3: 제보 거절

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | PENDING 제보 행의 "거절" 버튼 클릭 | 거절 확인 모달 표시 |
| 2 | 거절 확인 | 해당 제보의 상태가 REJECTED로 변경, 목록 갱신 |
| 3 | ACTIVE 상태 제보의 승인/거절 버튼 | 버튼 비활성화 또는 미표시 확인 |

### 시나리오 4: 공공데이터 동기화 트리거

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/sync` 접근 | 동기화 페이지 표시, 마지막 동기화 시각 표시 |
| 2 | "지금 동기화" 버튼 클릭 | 버튼 비활성화 + 로딩 표시 |
| 3 | 동기화 완료 (API mock 또는 실제 응답) | 완료 알림 표시, 마지막 동기화 시각 갱신 |
| 4 | 동기화 중 중복 클릭 시도 | 버튼 비활성화로 중복 요청 방지 |

### 시나리오 5: 화장실 목록 / 수정

| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | `/toilets` 접근 | 화장실 목록 테이블 표시 |
| 2 | 키워드 검색 입력 | 필터링된 결과 표시 |
| 3 | 수정 버튼 클릭 → 수정 폼 입력 | 입력값 검증 후 PATCH 요청 |
| 4 | 삭제 버튼 클릭 | 확인 모달 → 확인 시 DELETE 요청, 목록에서 제거 |

## 버그 등급 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| Critical | 서비스 불가 수준 | 로그인 불가, 제보 승인/거절 불가 |
| High | 핵심 기능 오작동 | 승인 후 상태 미갱신, 인증 가드 우회 가능 |
| Medium | 기능 일부 불편 | 페이지네이션 오작동, 에러 메시지 미표시 |
| Low | UI/UX 이슈 | 레이아웃 깨짐, 로딩 스피너 미표시 |

## QA 완료 기준 (GATE 4)

- Critical 버그 0건
- High 버그 0건
- 시나리오 1~4 전체 통과
- 인증 가드 우회 불가 확인

GATE 4 통과 시 오케스트레이터에게 다음 형식으로 결과 반환:

```
## QA 결과

- 시나리오 통과: X/5
- Critical: 0건
- High: 0건
- Medium: N건 (목록)
- Low: N건 (목록)

GATE 4: PASS / FAIL
```
